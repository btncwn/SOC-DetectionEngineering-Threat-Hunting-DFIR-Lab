# Detection Engineering with Sigma

## Overview

This section focuses on developing detection logic using Sigma rules and validating detections against telemetry collected throughout the lab environment.

The objective is to transform threat hunting findings, threat intelligence indicators, and DFIR investigations into reusable detections that can be deployed across multiple SIEM platforms.

## Objectives

* Develop Sigma detection rules
* Map detections to MITRE ATT&CK
* Convert Sigma rules to Splunk SPL
* Validate detections against lab telemetry
* Improve detection coverage
* Support SOC monitoring operations

## Planned Projects

### 01 - PowerShell Encoded Command Detection

Detection of suspicious PowerShell execution involving encoded commands.

### 02 - Suspicious Parent-Child Process Detection

Detection of abnormal process relationships commonly observed during attacks.

### 03 - Network Discovery Detection

Detection of host and service enumeration activity.

### 04 - Lateral Movement Detection

Detection of techniques associated with internal movement across systems.

### 05 - Threat Intelligence Driven Detection

Detection logic derived from threat intelligence indicators and investigative findings.
